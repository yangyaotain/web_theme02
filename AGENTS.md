# 数据要素平台静态原型约定

本项目是以根目录多 HTML 页面为主的静态原型，页面覆盖门户、数据社区、用户中心、运维中心、全要素展示驾驶舱等模块。修改时必须保持现有绿色主题、字体、字号、间距、按钮、菜单和卡片风格统一。

项目结构：
- 根目录 `*.html`：各业务页面。
- `css/`：公共和页面样式，优先复用已有变量和公共类。
- `js/`：公共交互和页面脚本。
- `images/`：图片资源。
- `all-elements-dashboard/`：全要素展示驾驶舱相关页面和资源。

原型修改规则：
- 先找同模块已有页面和公共菜单写法，例如运维中心、数据社区、用户中心、驾驶舱页面，再做增量修改。
- 运维中心只处理用户指定的菜单或页面时，不删除其他菜单；“默认进入某页面”不等于移除其他入口。
- 绿色主题下的高亮和告警配色要协调，不随意使用突兀紫色或与平台不一致的色系。
- 大屏类页面按 `1920×1080` 比例设计，但浏览器不是该比例时要允许自适应和必要的纵向滚动，避免底部大空白或内容被压缩变形。
- 示例数据要贴合数据交易、咨询服务、告警管理、站内消息、数据社区等当前业务语境。

执行边界：
- 用户说“先分析/确认后执行”时，只输出理解和方案，不写文件。
- 用户说“只完成列表”“只处理菜单”“新增和编辑后面设计”时，严格按范围做，不提前扩展。
- 默认不启动服务、不打开浏览器、不截图，除非用户明确要求。

检查：
- 默认运行 `git diff --check`。
- 修改 JS 后做轻量语法检查；系统 `node` 不可用时使用 Codex bundled Node。
- 中文显示异常时先用 UTF-8 重读确认。
